# unet_modeling1 파이프라인 및 주요 구현 설명

이 폴더(`unet_modeling1`)는 오디오 노이즈 제거(Denoising)를 위한 U-Net 모델의 초기 학습 스크립트(`02_unet_model.py`)를 포함하고 있습니다.

---

## 🔥 주요 구현 및 수정 사항 (핵심 강조)

이 스크립트에서는 안정적인 모델 학습과 직관적인 평가를 위해 **다음 두 가지 핵심 기능이 집중적으로 추가/수정**되었습니다.

1. **[중요!] 체크포인트(Checkpoint) 저장 및 이어서 학습(Resume) 기능 적용**
   - 긴 시간이 소요되는 모델 학습 중 예기치 않게 종료되는 상황을 방지하기 위해 **체크포인트 시스템이 도입**되었습니다.
   - 매 에폭이 끝날 때마다 모델 가중치(state_dict), 옵티마이저 상태, 현재 에폭, 최고 검증 로스(best_val_loss)를 `modeling1_checkpoint.pth` 파일에 덮어써서 저장합니다 (Last Checkpoint).
   - 스크립트 시작 시 해당 파일이 발견되면, **이전 학습 상태를 완벽히 복원하여 멈춘 지점부터 자동으로 이어서 학습을 재개**합니다.
   - 이와 별개로, 검증 성능(Validation Loss)이 갱신될 때마다 최고 성능의 모델은 `modeling1_best_unet_model.pth`라는 이름으로 안전하게 분리 저장됩니다.

2. **[중요!] 검증(Test/Validation) 과정에서 L1 Loss 단독 적용**
   - 검증 과정에서 복잡한 평가 지표를 섞어 쓰지 않고, 직관적이고 뚜렷한 스펙트로그램 값의 차이를 확인하기 위해 **오직 `L1 Loss` (Mean Absolute Error)만을 사용해 평가**하도록 통일했습니다.
   - 모델이 예측한 마스크를 노이즈 데이터에 곱해 추출한 `output`과 실제 깨끗한 음원의 스펙트로그램인 `clean` 간의 절대 오차를 바로 계산(`criterion(output, clean)`)하여, 모델의 잡음 제거 능력을 아주 직관적으로 모니터링할 수 있도록 수정되었습니다.

## 🚀 전체 학습 파이프라인 흐름 (`02_unet_model.py`)

전체 코드는 다음과 같은 흐름으로 오디오 스펙트로그램을 학습합니다.

1. **환경 설정 및 시드 고정 (`set_seed`)**
   - 실험의 완벽한 재현성을 보장하기 위해 파이토치, 넘파이, 파이썬 기본 난수 생성기 및 DataLoader의 워커(Worker) 시드까지 모두 42로 고정합니다.
2. **데이터 로드 (`get_dataloaders`)**
   - 상위 폴더의 `dataloader_utils`를 통해 LibriSpeech(깨끗한 음성)와 UrbanSound8K(노이즈) 데이터를 섞어 배치(Batch) 단위로 불러옵니다.
3. **모델 및 옵티마이저 초기화**
   - `UNet` 구조를 GPU(또는 CPU)에 올리고, 학습률 `0.0001`을 가지는 `Adam` 옵티마이저와 `L1Loss`를 정의합니다.
4. **학습 루프 (Train Loop)**
   - 모델의 순전파를 통해 나온 마스크(Mask) 결과로 손실(Loss)을 계산하고, 역전파하여 가중치를 업데이트합니다. (10 배치마다 로스 출력)
5. **검증 루프 (Validation Loop)**
   - 역전파를 비활성화(`torch.no_grad()`)한 채로 검증 데이터셋을 통과시키며 **평균 L1 Loss**를 도출합니다.
6. **결과 저장**
   - 위에서 강조한 체크포인트 로직을 거쳐 파일로 모델을 안전하게 저장하고 다음 에폭으로 넘어갑니다.
